# نظام تقارير اليومية — دليل النشر والتشغيل

تطبيق ويب أحادي الملف (index.html) لإدارة تقارير المبيعات والنقد اليومية للفروع، مع:
- **PWA** قابل للتثبيت والعمل دون إنترنت.
- **مزامنة سحابية** عبر Dropbox (نفس البيانات على كل أجهزتك).
- **تحديثات سلسة** دون فقدان أي بيانات، ونظام **ترحيل تلقائي** لبنية البيانات.

---

## 1) الملفات المطلوبة للنشر

ارفع كل هذه الملفات إلى جذر الاستضافة:

| الملف | الوظيفة |
|------|---------|
| `index.html` | التطبيق كاملاً |
| `sw.js` | عامل الخدمة (Service Worker): كاش، عمل دون إنترنت، إدارة التحديثات |
| `manifest.webmanifest` | بيان تطبيق الويب (اسم/أيقونات/ألوان التثبيت) |
| `icon.svg`, `icon-192.png`, `icon-512.png`, `icon-180.png` | الأيقونات (تثبيت + iOS + favicon) |
| `version.json` | رقم الإصدار الحالي (للتحقق من التحديثات) |
| `.nojekyll` | يمنع GitHub Pages من تجاهل الملفات |

> جميع المسارات **نسبية** (`./`)، فالتطبيق يعمل من أي نطاق أو مجلد فرعي بلا تعديل.

---

## 2) النشر على GitHub Pages (مجاني، موصى به)

1. أنشئ مستودعاً جديداً على GitHub (مثلاً `daily-report`).
2. ارفع كل الملفات أعلاه إلى الفرع `main`.
3. Settings ← Pages ← Source: `Deploy from a branch` ← Branch: `main` / `root` ← Save.
4. بعد دقيقة يصبح الرابط: `https://<اسم-المستخدم>.github.io/daily-report/`.
5. افتح الرابط على الجوال ← «إضافة إلى الشاشة الرئيسية» لتثبيته كتطبيق.

> **مهم:** خدمة العامل والـ PWA تعملان فقط عبر **HTTPS** (وهو مفعّل تلقائياً على GitHub Pages).

أي استضافة ساكنة أخرى (Netlify, Cloudflare Pages, Vercel, أو أي خادم HTTPS) تعمل بنفس الطريقة: ارفع الملفات كما هي.

---

## 3) المزامنة السحابية عبر Dropbox (للعمل على عدة أجهزة)

1. من التطبيق: تبويب **البيانات ← ربط حساب دروبوكس**، وأدخِل الـ App Key.
2. إنشاء الـ App Key: `dropbox.com/developers` ← Create app ← Scoped access ← App folder، فعّل صلاحيات `files.content.read` و`files.content.write`، وأضِف رابط موقعك في Redirect URIs، ثم انسخ App Key.
3. بعد الربط:
   - أي تعديل يُرفع تلقائياً إلى دروبوكس (مجلد `DailyReport/data.json`).
   - عند فتح التطبيق على جهاز آخر، يسحب أحدث نسخة تلقائياً.
   - يزامن أيضاً عند **العودة إلى التطبيق** أو **عودة الاتصال** أو كل 5 دقائق.
   - **حل التعارض:** الأحدث يفوز (ختم زمني `rev`)، فلا تدهس نسخةٌ أقدمُ نسخةً أحدث.

البيانات محفوظة محلياً على كل جهاز (IndexedDB/localStorage) وتعمل دون إنترنت، ودروبوكس هو طبقة المزامنة بين الأجهزة.

---

## 4) إصدار تحديث جديد (بدون فقدان بيانات، بدون إعادة تثبيت)

عند وجود نسخة جديدة من التطبيق:

1. عدّل رقم الإصدار في **ثلاثة مواضع** ليتطابق:
   - `index.html` → المتغيّر `APP_VERSION`
   - `sw.js` → الثابت `APP_VERSION`
   - `version.json` → الحقل `version`
2. ارفع الملفات المحدّثة إلى الاستضافة (استبدال المِلفّات القديمة فقط — لا تحذف شيئاً).
3. على جهاز المستخدم: يكتشف التطبيق التحديث تلقائياً ويعرض شريط **«تحديث الآن»**. بالضغط عليه يُطبَّق فوراً.
   - لا حاجة لإلغاء التثبيت أو إعادة التثبيت.
   - **كل البيانات والإعدادات والملفات المستوردة تبقى كما هي** (التحديث يستبدل ملفات الواجهة فقط، لا يمسّ قاعدة البيانات).

### تغييرات بنية البيانات (Migrations)
إذا غيّرت شكل البيانات في نسخة جديدة:
1. ارفع `SCHEMA_VERSION` في `index.html`.
2. أضِف دالة ترحيل جديدة إلى مصفوفة `MIGRATIONS` (الدالة رقم `v-1` ترقّي من النسخة `v` إلى `v+1`).
3. عند فتح التطبيق، تُطبَّق الترحيلات المتسلسلة تلقائياً على البيانات المحلية، وعلى أي بيانات أقدم تُسحب من دروبوكس، وعلى أي نسخة احتياطية تُستعاد — **دون فقدان بيانات**.

مثال دالة ترحيل:
```js
// 4 → 5: إضافة حقل جديد لكل يوم
function(st){ Object.keys(st.data).forEach(function(dt){ st.data[dt].newField = st.data[dt].newField || 0; }); }
```

---

## 5) النسخ الاحتياطي اليدوي

- **البيانات ← حفظ نسخة**: يُنزّل ملف JSON كامل (إعدادات + بيانات).
- **البيانات ← استعادة**: يستبدل البيانات من ملف JSON (مع ترحيل تلقائي إن كان الملف من نسخة أقدم).
- يمكن أيضاً حفظ نسخة داخل دروبوكس من قسم المزامنة.

---

## 6) بنية النظام باختصار

```
الاستضافة (GitHub Pages/HTTPS)  →  ملفات ثابتة: index.html + sw.js + manifest + icons
        │
        ├── PWA + Service Worker  →  تثبيت + عمل دون إنترنت + تحديثات سلسة
        │
        ├── تخزين محلي (IndexedDB/localStorage)  →  بيانات كل جهاز، سريعة، دون إنترنت
        │
        └── Dropbox (data.json)   →  مزامنة بين الأجهزة (الأحدث يفوز)
```

- **بسيط:** ملفات ثابتة فقط، بلا خادم أو قاعدة بيانات خلفية.
- **موثوق:** البيانات محلية أولاً، والسحابة نسخة مزامنة.
- **قابل للصيانة والنمو:** ترقيم إصدارات + ترحيل تلقائي + كاش مُصدَّر.
