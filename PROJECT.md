## 1) هوية المشروع
- تطبيق ويب عربي RTL **بملف واحد** (`index.html`) لتقارير اليومية لمحلات أثاث (شركة الضبيبي التجارية، السعودية): مبيعات فروع، طرق دفع، مصروفات/إيرادات، رصيد مرحّل، طباعة رسمية، استيراد/تصدير إكسل.
- المالك **غير تقني** — كل التواصل بالعربية، وكل تعديل يُسلَّم كملفات جاهزة للرفع + حزمة zip.
- النشر: GitHub Pages — مستودع `dhtrd/daybook` → `dhtrd.github.io/daybook`. تحديث = رفع `index.html` + `sw.js` + `version.json` (الأرقام الثلاثة يجب أن تتطابق دائماً).
- المزامنة: Cloudflare Worker `daybook-sync.storedh2024.workers.dev` ↔ Dropbox (`/DailyReport/data.json` + نسخ مؤرخة في `/DailyReport/backups/`). إعداد العميل في `config.js` (cloudUrl فقط، لا أسرار).

## 2) الإصدار الحالي والملفات
- **الإصدار: 1.4.3** · schema: 5.

## 3) الثوابت المحاسبية — لا تُمس أبداً
- نقد الصف = المبيعات − مجموع طرق الدفع غير النقدية.
- باقي الرصيد = رصيد سابق + نقد اليوم − الإيداع البنكي (سلسلة مترابطة عبر الأيام؛ `carryIn` يثبّت الرصيد السابق يدوياً عند الحاجة).
- الصافي النقدي = النقد − المصروفات · الإجمالي النقدي = الصافي + الإيرادات.
- `balances()` تحسب السلسلة كاملة لكل الأيام (تشمل السنوات «المخفية» بالأرشفة).

## 4) معمارية المزامنة (الأهم)
- محلي: IndexedDB (`DB`)؛ الحالة في كائن `S`؛ الحفظ عبر `commitDay/persistDay/persistEverything`.
- **محرك العمليات**: كل تعديل يُسجَّل في `S.config.pendingOps` (سجل لكل يوم/مفتاح إعداد) مع أساس `baseSnap` + `baseRev`. السحب يدمج: يجلب السحابة، يعيد تطبيق العمليات فوقها (`replayOps`)، تعارض نفس السجل فقط = قرار مستخدم.
- **عقد الـWorker**: `GET /data` → الحمولة + ترويسة `X-Data-Rev` (204 إن لا ملف) · `POST /data` → الجسم = الحمولة كاملة + `X-Base-Rev` (قفل تفاؤلي؛ 409 = تعارض) · `POST /backup` → نسخة مؤرخة.
- **حمولة السحابة** (`gatherPayload`): `{app:"dsr",version:3,schema,appVersion,epoch,rev,exportedAt,config,data}` — الأسرار تُنظَّف (refreshToken فارغ)، وحقول المزامنة تُحذف من config.
- **نظام العهد السيادي (v1.4.3)** — درع مضاد للمسح الجماعي:
  - كل حمولة تحمل `epoch`. الفرض (استعادة نسخة / rescue / «فرض المحلي») يرسل ترويسة `X-Sovereign:1` مع `epoch = الحالي+1`.
  - الخادم يرفض 409: كتابة عادية بعهد مختلف (`stale`) · كتابة عادية تحذف >15 يوماً أو تمحو سنة قائمة ≥20 يوماً (`guard`) · فرضاً سيادياً لا يرفع العهد (`sovereign`).
  - العميل عند `stale/guard` أو عند سحبِ عهدٍ أعلى → **اعتماد تلقائي** (`adoptCloud`): يعتمد السحابة كاملة، يفرّغ طابوره، توست «اعتُمدت النسخة المُستعادة مركزياً ✓» — الجهاز العالق يُطهَّر ويكمل صاحبه عمله.
- «أدوات إنقاذ المزامنة» في صفحة البيانات: «مزامنة نظيفة من السحابة» (يعتمد السحابة ويهمل المعلق المحلي) و«فرض المحلي على السحابة» (سيادي). و«استعادة نسخة» تفرض سيادياً تلقائياً.
